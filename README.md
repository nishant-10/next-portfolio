# Nishant Joshi — Portfolio

Personal portfolio for Nishant Joshi (Software Engineer), built with
[Next.js](https://nextjs.org) (App Router) and React 19, deployed on Vercel to
[nishantjoshi.com](https://www.nishantjoshi.com).

## Architecture

- **`app/`** — the Next.js App Router entry.
  - `layout.js` — the root layout: `<html>`/`<body>`, the looping background
    video, all SEO metadata (Metadata API), and JSON-LD `Person` structured
    data. This is a **server component**, so the metadata is rendered on the
    server.
  - `page.js` — a **server component** that composes the page sections.
  - `sitemap.js`, `robots.js`, `manifest.js` — Next.js metadata routes that
    generate `/sitemap.xml`, `/robots.txt` and `/manifest.webmanifest`.
  - `globals.css` — global styles (background video, scrollbar, focus states).
- **`src/components/`** — the section components.
  - Only `navbar.js` and `ProjectCards.js` are client components (`"use client"`)
    because they use interactive state. Everything else is a server component,
    so the content is server-rendered and fully crawlable.
  - `Icons.js` holds the shared social/UI SVGs.
- **`src/data/home_content.json`** — all copy, socials and projects. Edit this
  file to update the site; no component changes required.
- **`src/css/`** — per-section stylesheets.

## SEO

- **Metadata API** (`app/layout.js`) — title template, description, keywords,
  canonical URL, Open Graph + Twitter Card tags, robots directives, theme color
  and icons.
- **JSON-LD** `Person` structured data (job, employer, education, skills,
  social profiles).
- **`sitemap.xml`, `robots.txt`, `manifest.webmanifest`** — generated by the
  metadata routes in `app/`.
- **Static prerendering** — the page is generated at build time (`○ Static`),
  so crawlers and social scrapers receive fully-rendered HTML.
- **Semantic, accessible markup** — `header`/`main`/`section`/`footer`, a single
  `<h1>`, ordered `h2`/`h3` headings, `alt` text, `aria-label`s and
  `next/image` for the photo.
- **Google Search Console** is verified via
  `public/google802413fa5ab93c6b.html`.

> When the domain or content changes, update the URLs in `app/layout.js`
> (`metadataBase`, Open Graph), `app/sitemap.js` and `app/robots.js`. For the
> best social previews, add a dedicated 1200×630 image and point `openGraph`
> and `twitter` images at it (currently `public/compress3.jpg`, 500×500).
